# Implementation Plan: Fix Bot Rematch Auto-Accept and Position Swap

**Branch**: `001-fix-bot-rematch-swap` | **Date**: 2026-05-19 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-fix-bot-rematch-swap/spec.md`

## Summary

When a human plays against a bot and rematches, the bot must auto-accept regardless of which seat (
player one or player two) it occupies in the rematch session. Currently, `RematchSession` only
handles bot-as-player-two; after the first rematch the bot becomes player one and the session never
confirms. The fix adds a symmetric `playerOneIsBot` field to the domain aggregate, the JPA entity,
and the related event handler and notification translator.

## Implementation Steps

### Step 1 — DB Migration

Create `V17__add_player_one_is_bot.sql`:

```sql
ALTER TABLE rematch_sessions
    ADD COLUMN player_one_is_bot BOOLEAN NOT NULL DEFAULT FALSE;
```

No index needed. Default `FALSE` is correct for all existing rows.

### Step 2 — Domain: `RematchSessionOpenedEvent`

Add `playerOneIsBot` field (boolean) with getter. Update constructor to accept it. Existing
`playerTwoIsBot` field and all existing callers remain unchanged.

### Step 3 — Domain: `RematchSession`

Three sub-changes:

**3a. Field + constructor**: Add `boolean playerOneIsBot` field. Update the private constructor and
`reconstruct()` factory to accept it. Keep `playerTwoIsBot` as-is.

**3b. `open()` factory**: Determine `playerOneChoice` symmetrically:

```java
final var p1Choice =
    playerOneIsBot ? RematchPlayerChoice.WANTS_REMATCH : RematchPlayerChoice.UNDECIDED;
final var p2Choice =
    playerTwoIsBot ? RematchPlayerChoice.WANTS_REMATCH : RematchPlayerChoice.UNDECIDED;
```

Update the `RematchSessionOpenedEvent` emission to pass `playerOneIsBot`.

**3c. `chooseRematch()` and `leave()`**: Add symmetric bot guard:

```java
// chooseRematch guard (add before existing p2 guard):
if (isPlayerOne(actor) && playerOneIsBot) { return; }

// leave guard (extend existing):
if (!isParticipant(actor)) { throw new NotParticipantOfRematchSessionException(); }
if (playerTwoIsBot && !isPlayerOne(actor)) { throw new BotCannotLeaveRematchSessionException(); }
if (playerOneIsBot && isPlayerOne(actor)) { throw new BotCannotLeaveRematchSessionException(); }
```

Add `isPlayerOneIsBot()` getter.

### Step 4 — Application: `MatchFinishedRematchSessionCreator`

Query the bot registry for player one as well:

```java
final var playerOneIsBot = botRegistry.isBot(playerOneId);
final var playerTwoIsBot = botRegistry.isBot(playerTwoId);

RematchSession.open(matchId, playerOneId, playerTwoId, gamesToWin,
    playerOneIsBot, playerTwoIsBot, clock.instant(), rematchDuration);
```

Note: `RematchSession.open()` signature changes to accept `playerOneIsBot` before `playerTwoIsBot`.

### Step 5 — Application: `RematchNotificationEventTranslator`

Fix `handleOpened()` to exclude the bot from recipients regardless of seat:

```java
private void handleOpened(final RematchSessionOpenedEvent event) {
    final List<PlayerId> recipients = new ArrayList<>();
    if (!event.isPlayerOneIsBot()) {
        recipients.add(event.getPlayerOneId());
    }
    if (!event.isPlayerTwoIsBot()) {
        recipients.add(event.getPlayerTwoId());
    }
    // ... build payload and publish unchanged
}
```

### Step 6 — Infrastructure: JPA Entity + Mapper

**`RematchSessionJpaEntity`**: Add
`@Column(name = "player_one_is_bot", nullable = false) private boolean playerOneIsBot` with
getter/setter.

**`RematchSessionMapper`**:

- `toEntity()`: `entity.setPlayerOneIsBot(session.isPlayerOneIsBot())`
- `toDomain()`: pass `entity.isPlayerOneIsBot()` to `RematchSession.reconstruct()`

### Step 7 — Tests

**`RematchSessionTest`** — add to the `Open` nested class:

- `sets playerOne choice to WANTS_REMATCH when playerOne is bot`
- `bot playerOne starts with WANTS_REMATCH; playerTwo starts UNDECIDED`

Add to the `ChooseRematch` nested class:

- `bot match with bot as player one confirms immediately when playerTwo (human) chooses`
- `bot as player one: calling chooseRematch with bot actor is a no-op`

Add to the `Leave` nested class:

- `throws BotCannotLeaveRematchSessionException when bot as player one tries to leave`

**`ChooseRematchCommandHandlerTest`** — add:

- `confirms session immediately when bot is player one and human (player two) chooses`
